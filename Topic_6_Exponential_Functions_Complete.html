Bonus Question 1: (1) 3⁶
Solution: When multiplying powers with the same base, add the exponents: 3⁴ · 3² = 3⁴⁺² = 3⁶
Bonus Question 2: (3) 200
Solution: In the exponential function P(t) = 200(1.15)ᵗ, the coefficient 200 represents the initial population when t = 0.
Bonus Question 3: (3) 1/8
Solution: After 60 years, there are 60/20 = 3 half-lives. After each half-life, half remains: (1/2)³ = 1/8
Bonus Question 4: (2) y = 500(0.8)ˣ
Solution: Exponential decay occurs when the base is between 0 and 1. Only option (2) has a base of 0.8, which is less than 1.
Bonus Question 5: (3) 32
Solution: If log₂ x = 5, then 2⁵ = x, so x = 32
Bonus Question 6: (1) 6%
Solution: The rule of 72 states that doubling time ≈ 72/rate. If doubling time is 12 years, then 12 = 72/rate, so rate = 72/12 = 6%

SUCCESS STRATEGIES FOR EXPONENTIAL FUNCTIONS

For Students:

  • Show ALL work: Even for multiple choice, show your calculation steps
  • Check your work: Substitute answers back into original equations
  • Use proper notation: Distinguish between 2x (linear) and 2ˣ (exponential)
  • Master exponent rules: Product rule, quotient rule, and power rule
  • Understand growth vs. decay: Base > 1 is growth, 0 < base < 1 is decay
  • Practice with calculator: Know how to enter exponents correctly on TI-84

Common Student Errors to Avoid:

  • Confusing exponential and polynomial: 2ˣ ≠ x²
  • Incorrect exponent rules: (aᵐ)ⁿ = aᵐⁿ, NOT aᵐ⁺ⁿ
  • Forgetting parentheses: 2·3ˣ on calculator needs to be entered as 2*(3^x)
  • Misidentifying initial value: In f(x) = ab^x, 'a' is initial value, not 'b'
  • Wrong asymptote: y = 2ˣ has horizontal asymptote at y = 0, not x = 0
  • Incorrect log conversion: 2⁵ = 32 means log₂ 32 = 5, not log₃₂ 2 = 5

Time Management Tips:

  • MCQ (1-14): Aim for 1-2 minutes per question
  • Extended Response (15-20): Allocate 5-8 minutes per question
  • Priority Order: Complete familiar exponential forms first, save complex applications for last
  • Calculator Efficiency: Set up Y= functions early for quick reference and graphing

Verification Strategies:

  1. Substitution Check: Plug solutions back into original equation to verify
  2. Graphical Confirmation: Graph both sides of equation and verify intersection points
  3. Base Analysis: Verify growth/decay by checking if base is greater than or less than 1
  4. Asymptote Check: Confirm horizontal asymptote by evaluating limits
  5. Table Verification: Use TABLE feature to verify exponential pattern (constant ratio)
  6. Logarithm Reversal: Convert between exponential and logarithmic forms to verify

Real-World Application Categories:

  • Population Growth: Bacteria, animals, human populations with constant growth rate
  • Financial Mathematics: Compound interest, investment growth, loan calculations
  • Radioactive Decay: Half-life problems, carbon dating, medical isotopes
  • Temperature Change: Newton's Law of Cooling, heating processes
  • Medicine: Drug concentration decay in bloodstream
  • Technology: Moore's Law (computer processing power), viral social media growth

Pattern Recognition Skills:

  • Exponential Growth: Base b > 1, y-values increase rapidly, graph curves upward
  • Exponential Decay: Base 0 < b < 1, y-values decrease toward zero, graph curves downward
  • Y-intercept: Always equals 'a' in f(x) = ab^x when x = 0
  • Horizontal Asymptote: For f(x) = ab^x + c, asymptote is y = c
  • Doubling Time: Time for quantity to double is constant in exponential growth
  • Half-Life Pattern: After n half-lives, fraction remaining is (1/2)^n

  • Change of Base: Use log_b(x) = log(x)/log(b) for calculator
  • Graph Interpretation: Identify growth/decay from graph shape and behavior
  • Real-World Modeling: Translate word problems into exponential equations
